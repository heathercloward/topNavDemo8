Configure Data Download During the Final Finish Process to use a CranPort Package

If the Finish Type is CranPort, then a corresponding Table definition must exist in Collect. Refer to Tables in Collect for more information.

NOTE: dspConductâ„¢ does not confirm that the table exists when registering a table on the Task Final Finish Tables page. An error displays during the Final Finish if the table does not exist.

NOTE: Refer to Final Finish Process for a Request Overview and Final Finish Setup Example for general information.

To register a CranPort package to download a table for the Final Finish process:

  1. Click dspConduct >Design in the Navigation pane.
  2. Click the Tasks icon for a category.
  3. Click the Final Finish Tables icon for a main parent task (i.e., Request (Address â€“ General), as opposed to Request (Address â€“ Telephone/Mobile)).
  4. It is recommended that tables should not be registered to a child task.

  5. Click Add.

    View the field descriptions for the Task Final Finish Tables page

  6. Select CranPort from the FINISH TYPE list box.

  7. Enter a number in the PRIORITY field.

    NOTE: The execution order of packages is by priority of role (specified on the Role page) to which the task is assigned, then priority order specified on the Task Final Finish Tables page.  For example, the priority for Role A set on the Roles page is 10. Role A has 3 tables registered on the Task Final Finish page with a priority of 10, 20, and 30.  The priority for Role B set on the Roles page is 20. Role B has 2 tables registered on the Task Final Finish Tables page with a priority of 10 and 20. The execution order will be:
    Role A (priority 10), table with priority 10, then table with priority 20, then table with priority 30
    Then
    Role B (priority 20), table with priority 10, then table with priority 20

  8. Click Save; Vertical View displays.
  9. Select a Collect source from the Source list box.
  10. Select a Collect target from the Target list box.
  11. Select the name of the table to download from the Table Name list box.

  • Select an option in the Run Rules After Download list box, if applicable.

    NOTE" This option determines the Collect rules to run on a table after download, when a user with the Post role clicks Finish for the request on the Request Role page. Options are:

    • All â€“ Run all of the rules:  If multiple rules are registered to the table in Collect, dspConductâ„¢ runs them on the table after download in the priority order set on the Table (Rule) page in Collect.

    None â€“ Do not run any rules: Though rules are registered to the table in Collect, dspConductâ„¢ does not run those rules on the downloaded table.

    NoParametersOnly â€“ Run only Collect rules that do not contain input parameters, such as an Insert rule.

    ParametersOnly â€“ Run Collect rules that contain input parameters, such as an Update rule.

    NOTE: The rule(s) must have already been registered and activated in Collect.

  1. Click Save.
